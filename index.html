Quick Start

Up and running in 60 seconds

Works with OpenAI, Anthropic, Langchain, AutoGen, CrewAI, and any custom agent framework.

1. Install

pip install agentsentinel Python
npm install @agentsentinel/sdk Node.js

2. What you get instantly

  • ✓ Real-time cost dashboard
  • ✓ Slack / email alerts
  • ✓ Per-agent spend breakdown
  • ✓ Approval request queue
  • ✓ Exportable audit logs
my_agent.py 
from agentsentinel import AgentGuard, AgentPolicy

# 1. Define your safety policy
policy = AgentPolicy(
    daily_budget=10.00,       # max $10/day
    hourly_budget=2.00,      # max $2/hour
    require_approval=[
        "delete_*",           # any delete action
        "send_email",         # sending comms
        "execute_sql",        # DB writes
    ],
    alert_channel="slack",
)

# 2. Wrap your agent
guard = AgentGuard(policy=policy)

# 3. Protect your tools with a decorator
@guard.protect
def send_email(to, subject, body):
    # AgentSentinel pauses here and asks
    # for human approval before sending
    email_client.send(to, subject, body)

FAQ

Everything you want to know

Do I need to change my infrastructure? +

No. AgentSentinel is a pure Python/Node.js decorator layer. You install the package, wrap your tool functions, and that's it. No proxies, no sidecars, no Kubernetes operators. It works in a Lambda, a Docker container, or a local script.

Does this work with Langchain, AutoGen, or CrewAI? +

Yes. AgentSentinel wraps at the tool/function level, which is framework-agnostic. We have first-class integrations with Langchain tools, AutoGen functions, and CrewAI tasks, plus a generic decorator that works with anything else.

How accurate is the cost tracking? +

Very. We intercept token counts from the model API responses and apply the current provider pricing tables (updated daily). In our tests against actual OpenAI invoices, we're within 1–2% — close enough to trust for budget enforcement.

What happens when a budget limit is hit? +

By default, the agent raises a BudgetExceededError and stops execution. You can configure it to: (a) pause and wait for human approval to continue, (b) send a notification and keep running at a reduced rate, or (c) hard-stop and alert your on-call.

Can I self-host the dashboard? +

Yes, on the Enterprise plan. We provide a Docker Compose stack and a Helm chart for Kubernetes. The SDK sends events to your self-hosted instance instead of our cloud. The SDK itself is open source, so you can verify exactly what data is collected.

Do you see my prompts or data? +

By default, we log metadata only: timestamps, token counts, tool names, and cost estimates. We never log prompt content or tool arguments unless you opt in to full logging for debugging. All data is encrypted in transit and at rest, and we are SOC 2 Type II certified.
